Найденные антипаттерны:
- использование модификатора private, замена где есть вероятность последующего наследования на protected
- некоторые классы можно убрать, переместив их бизнес-логику в подходящий класс
- не реализована проверка входных значений (слепая вера)
- возможно, не оптимальная архитектура, решение - перепроектирование
